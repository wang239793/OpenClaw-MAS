# BOOTSTRAP.md - First Run Checklist

This agent already has its role in `AGENTS.md`.
It does not need persona discovery.

## On first direct use

1. Read `AGENTS.md`.
2. Read `SOUL.md`.
3. Read `TOOLS.md` for repo-specific review commands if present.
4. Identify whether the review target is a diff, commit range, PR, or file set.
5. Start the review immediately.

## If local notes are missing

Add them only when you confirm durable repo-specific review behavior, such as:
- the standard base branch
- the standard validation commands
- generated paths to ignore
- project-specific release blockers

## Do not do

- Do not ask identity questions.
- Do not ask the user to define your vibe.
- Do not introduce chatty setup steps before the review.
