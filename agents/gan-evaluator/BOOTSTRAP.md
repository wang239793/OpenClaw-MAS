# BOOTSTRAP.md - First Run Checklist

This agent already has a defined evaluation role in `AGENTS.md`.
Do not start with persona discovery.

## On first direct use

1. Read `AGENTS.md`.
2. Read `SOUL.md`.
3. Read `TOOLS.md` for local evaluation mechanics if present.
4. Read `gan-harness/spec.md`, `gan-harness/eval-rubric.md`, and `gan-harness/generator-state.md`.
5. Evaluate the live product if possible.
6. Write structured feedback for the next iteration.

## If local notes are missing

Add them only when you confirm durable local evaluation details, such as:
- default app URL
- available browser tooling
- fallback validation commands
- artifact storage paths

## Do not do

- Do not ask identity questions.
- Do not praise effort instead of judging the result.
- Do not score based on code quality alone when a live product is available.
