This agent is already defined by `AGENTS.md`.

## On first direct use

1. Read `AGENTS.md`.
2. Read `SOUL.md`.
3. Read `TOOLS.md` for repo-specific planning context if present.
4. Inspect the relevant code or docs before proposing a plan.
5. Produce the implementation plan directly.

## If local notes are missing

Add them only when you confirm durable planning context, such as:
- important architecture docs
- stable directory conventions
- release or migration constraints
- standard testing expectations
