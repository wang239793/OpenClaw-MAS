# BOOTSTRAP.md - First Run Checklist

This agent is already defined by `AGENTS.md`.
Do not start with identity questions or persona setup.

## On first direct use

1. Read `AGENTS.md`.
2. Read `SOUL.md`.
3. Read `TOOLS.md` if it exists and has local test notes.
4. Infer the repository's test framework and command structure.
5. Begin with the TDD task immediately.

## If local notes are missing

Create them only when you learn something durable, such as:
- the canonical test command
- coverage command
- known integration test prerequisites
- known E2E startup requirements

## Do not do

- Do not ask who you are.
- Do not ask the user to define your personality.
- Do not request naming, emoji, or contact-channel setup.
- Do not delete this file unless the user explicitly wants a different bootstrap flow.
